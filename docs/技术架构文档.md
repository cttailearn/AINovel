# 技术架构文档

> **版本**：v1.1.0
> **架构模式**：Tauri 桌面壳 + React 前端 + FastAPI 后端（前后端分离）

## 1. 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                  桌面应用（用户双击启动）                      │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Tauri 桌面窗口 (tauri://localhost)                  │  │
│  │  ┌────────────────────────────────────────────────┐  │  │
│  │  │  React 前端 (WebView2 渲染)                    │  │  │
│  │  │  - App.jsx (主容器、Tab 切换)                  │  │  │
│  │  │  - ModelConfigPanel.jsx                        │  │  │
│  │  │  - NovelPanel.jsx / NovelReader.jsx            │  │  │
│  │  │  - api/client.js (HTTP 客户端)                 │  │  │
│  │  └────────────────────────────────────────────────┘  │  │
│  │  ↑ fetch /api/*                                       │  │
│  │  ↑ CORS 允许: tauri://localhost                       │  │
│  └──────────────────────────────────────────────────────┘  │
│                       ↑ HTTP                                 │
│                       ↓                                       │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  FastAPI 后端 (Python 进程)                          │  │
│  │  - main.py (应用入口、CORS)                          │  │
│  │  - routers/ (API 路由)                              │  │
│  │  - services/ (业务逻辑)                             │  │
│  │  - database.py (SQLite 操作)                        │  │
│  └──────────────────────────────────────────────────────┘  │
│                       ↑                                       │
│                       ↓                                       │
│              ┌──────────────────┐                            │
│              │  SQLite 数据库    │                            │
│              │  backend/data/    │                            │
│              │  models.db        │                            │
│              │  novels/*.txt     │                            │
│              └──────────────────┘                            │
└─────────────────────────────────────────────────────────────┘
```

### 1.1 三层职责

| 层 | 职责 | 技术 |
|----|------|------|
| **展示层** | 渲染 UI、用户交互、调用 API | React 18 + Vite 5 |
| **应用层** | 业务逻辑、数据处理、API 服务 | FastAPI + Python 3.10+ |
| **数据层** | 数据持久化、文件存储 | SQLite + 本地文件系统 |

### 1.2 进程模型

| 进程 | 启动方式 | 端口 | 生命周期 |
|------|----------|------|----------|
| **Tauri exe** | 用户双击 | - | 关闭窗口即退出 |
| **FastAPI** | `start-app.bat` / Tauri sidecar | 8008 | 独立进程 |
| **WebView2** | Tauri 自动启动 | - | Tauri 退出时关闭 |

## 3. Tauri 集成

### 3.1 Tauri 工作原理

```
┌─────────────────────────────────────────┐
│  ai-novel-desktop.exe (Tauri Shell)     │
│                                          │
│  1. 加载 tauri.conf.json                 │
│  2. 读取 frontendDist (../frontend/dist)│
│  3. 启动 WebView2 进程                   │
│  4. 在 WebView 中显示 React 界面         │
│  5. 处理窗口事件（最小化/关闭）          │
└─────────────────────────────────────────┘
```

### 3.2 tauri.conf.json 关键配置

```json
{
  "productName": "AI小说管理",
  "identifier": "com.ainovel.desktop",
  "build": {
    "frontendDist": "../frontend/dist"  // 嵌入前端构建产物
  },
  "app": {
    "windows": [{
      "title": "AI 小说管理系统",
      "width": 1280,
      "height": 800
    }],
    "security": {
      "csp": null                       // 允许跨域 fetch
    }
  }
}
```

### 3.3 前端检测 Tauri 环境

`frontend/src/api/client.js`：
```javascript
const API_PREFIX = (typeof window !== 'undefined' && window.__TAURI_INTERNALS__)
  ? 'http://127.0.0.1:8008/api'   // Tauri 内：绝对 URL
  : '/api';                       // 浏览器：相对 URL（Vite 代理）
```

`backend/config.py`：

## 4. 后端架构

### 4.1 数据库设计

#### model_configs（AI 模型配置）
```sql
CREATE TABLE model_configs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    provider TEXT NOT NULL,
    model_url TEXT NOT NULL,
    api_key TEXT NOT NULL,
    model_name TEXT NOT NULL,
    enabled INTEGER DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### novels
```sql
CREATE TABLE novels (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    author TEXT DEFAULT '未知作者',
    filename TEXT NOT NULL,
    file_path TEXT NOT NULL,
    file_size INTEGER DEFAULT 0,
    status TEXT DEFAULT 'pending',
    parse_rule TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### chapters
```sql
CREATE TABLE chapters (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    novel_id INTEGER NOT NULL,
    chapter_number INTEGER NOT NULL,
    title TEXT NOT NULL,
    content TEXT,
    start_position INTEGER DEFAULT 0,
    end_position INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (novel_id) REFERENCES novels(id) ON DELETE CASCADE
);
```

## 7. 章节解析流程

### 7.1 算法

```
输入: 小说文件 + 正则规则
  ↓
1. 读取整个文件内容
  ↓
2. 用正则 re.finditer() 匹配所有章节标题
  ↓
3. 记录每个匹配的 start_position
  ↓
4. 切片提取每个章节内容:
     chapter_content = content[start:end]
  ↓
5. 批量插入 chapters 表
  ↓
6. 更新 novels.status = 'parsed'
  ↓
输出: 章节列表 + 内容
```

### 7.2 默认正则

```python
DEFAULT_RULES = [
    r"第.+?章",                                   # 通用中文
    r"第\d+章",                                   # 纯数字
    r"^第[一二三四五六七八九十百千零\d]+章.*",     # 中文数字
]
```

## 12. 故障排查

| 症状 | 原因 | 解决 |
|------|------|------|
| 启动后报"无法连接 8008" | FastAPI 没起 | 运行 `scripts\start-app.bat` |
| 前端白屏 | Tauri 找不到 dist | 重新 `npm run build` |
| 上传失败 | 文件 > 50MB | 调 `MAX_UPLOAD_SIZE` |
| 章节解析无结果 | 正则不匹配 | 测试 regex，确认文件编码 |
| Tauri 启动慢 | WebView2 首次初始化 | 二次启动会快 |
| cargo 编译失败 | 网络问题 | 已配 rsproxy 镜像 |
