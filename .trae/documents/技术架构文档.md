- **前端**：React 18.x + Vite 5.x

## 3. 后端架构

### 3.1 数据库设计

#### model_configs 表（AI 模型配置）
```sql
CREATE TABLE model_configs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    provider TEXT NOT NULL,
    model_url TEXT NOT NULL,
    api_key TEXT NOT NULL,
    model_name TEXT NOT NULL,
    enabled INTEGER DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### novels 表（小说信息）
```sql
CREATE TABLE novels (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    author TEXT DEFAULT '未知作者',
    filename TEXT NOT NULL,
    file_path TEXT NOT NULL,
    file_size INTEGER DEFAULT 0,
    status TEXT DEFAULT 'pending',
    parse_rule TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### chapters 表（章节信息）
```sql
CREATE TABLE chapters (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    novel_id INTEGER NOT NULL,
    chapter_number INTEGER NOT NULL,
    title TEXT NOT NULL,
    content TEXT,
    start_position INTEGER DEFAULT 0,
    end_position INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (novel_id) REFERENCES novels(id) ON DELETE CASCADE
);
```

### 3.2 核心模块设计

#### database.py
**职责**：
- 数据库初始化和连接管理
- AI 模型配置 CRUD 操作
- 小说 CRUD 操作
- 章节 CRUD 操作

**关键函数**：
- `init_db()`: 初始化数据库表
- `save_novel()`: 保存小说信息
- `get_novel_by_id()`: 获取单个小说
- `get_all_novels()`: 获取所有小说
- `delete_novel_by_id()`: 删除小说
- `save_chapter()`: 保存章节
- `get_chapters_by_novel()`: 获取小说的章节
- `get_chapter_content()`: 获取章节内容

#### main.py
**职责**：
- FastAPI 应用入口
- API 路由定义
- CORS 中间件配置

**路由结构**：
```
/api/health                    # 健康检查
/api/models/*                  # AI 模型配置
/api/novels                    # 小说列表
/api/novels/upload             # 上传小说
/api/novels/{id}               # 小说详情
/api/novels/{id}/parse         # 解析章节
/api/novels/{id}/parse-rule    # 设置解析规则
/api/novels/{id}/chapters/*    # 章节操作
```

## 5. 章节解析逻辑

### 5.1 正则规则
默认支持的规则模式：
1. `第.+?章` - 通用中文章节
2. `第\\d+章` - 纯数字章节
3. `^第[一二三四五六七八九十百千零\\d]+章.*` - 中文数字章节

### 5.2 解析流程
1. 读取小说文件内容
2. 使用正则表达式匹配所有章节标题
3. 记录每个章节的起始位置
4. 保存章节信息到数据库
5. 更新小说状态为 "parsed"

### 5.3 位置记录
- `start_position`: 章节标题在文件中的位置
- `end_position`: 下一章节开始位置（当前章节结束位置）
- 最后一个章节的 end_position 为文件大小

## 7. 运行说明

### 后端运行
```bash
cd backend
python main.py
```
- 默认端口：8008
- 地址：http://127.0.0.1:8008

### 前端运行
```bash
cd frontend
npm install
npm run dev
```
- 默认端口：5173
- 自动代理 /api 请求到后端
